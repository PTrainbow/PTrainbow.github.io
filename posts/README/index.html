AI 任务数据解耦方案

问题描述

原来的 getUnShownContentIdsForSQL 方法直接依赖 RecyclerView 的数据（mVideoGoodsViewListData 和 mTotalShowCurrentPosition），导致无法在 RecyclerView 外部直接调用 AI 任务。

解决方案

通过引入 AITaskDataManager 数据管理器，实现数据与 RecyclerView 的解耦，支持通过 taskKey 直接调用 AI 任务。

核心组件

1. AITaskDataManager

单例数据管理器，负责存储和管理每个 taskKey 对应的数据。

主要方法：

  • registerTaskData(taskKey, data, currentPosition): 注册任务数据
  • updateTaskPosition(taskKey, currentPosition): 更新任务位置
  • getUnShownContentIdsForSQL(taskKey): 获取未显示内容ID（SQL格式）
  • clearTaskData(taskKey): 清理任务数据

2. AITaskProcedure

AI 任务处理器，使用 AITaskDataManager 获取数据，不再直接依赖 RecyclerView。

3. DataReorderListener

接口，用于获取 taskKey。

使用方法

1. 在 RecyclerView 中注册数据

class VideoRecyclerViewAdapter {
    private val taskKey = "video_task_001"
    
    fun setData(data: VideoGoodsViewListData) {
        // 注册数据到管理器
        AITaskDataManager.getInstance().registerTaskData(
            taskKey = taskKey,
            data = data,
            currentPosition = currentPosition
        )
    }
    
    fun onPositionChanged(newPosition: Int) {
        // 更新位置信息
        AITaskDataManager.getInstance().updateTaskPosition(taskKey, newPosition)
    }
    
    fun clearData() {
        // 清理数据
        AITaskDataManager.getInstance().clearTaskData(taskKey)
    }
}

2. 直接通过 taskKey 调用 AI 任务

class AITaskCaller {
    suspend fun callAITaskByTaskKey(taskKey: String): List<String> {
        val taskProcedure = AITaskProcedure(object : DataReorderListener {
            override fun getAITaskId(): String = taskKey
        })
        return taskProcedure.startTask()
    }
}

// 使用示例
suspend fun example() {
    val taskKey = "video_task_001"
    val result = AITaskCaller().callAITaskByTaskKey(taskKey)
    println("AI 任务结果: $result")
}

3. 在 ViewModel 中使用

class VideoViewModel {
    private val taskKey = "video_task_001"
    
    suspend fun performAITask(): List<String> {
        return AITaskCaller().callAITaskByTaskKey(taskKey)
    }
    
    fun updateVideoData(data: VideoGoodsViewListData, currentPosition: Int) {
        AITaskDataManager.getInstance().registerTaskData(
            taskKey = taskKey,
            data = data,
            currentPosition = currentPosition
        )
    }
}

优势

  1. 解耦: AI 任务不再直接依赖 RecyclerView 的数据
  2. 灵活性: 可以在任意地方通过 taskKey 调用 AI 任务
  3. 可维护性: 数据管理集中化，便于维护和扩展
  4. 线程安全: 使用单例模式确保数据一致性
  5. 内存管理: 提供数据清理方法，避免内存泄漏

注意事项

  1. 数据同步: 确保 RecyclerView 中的数据变化及时同步到 AITaskDataManager
  2. taskKey 唯一性: 每个任务使用唯一的 taskKey，避免数据冲突
  3. 内存清理: 在适当的时机调用 clearTaskData 清理不需要的数据
  4. 错误处理: 在获取数据失败时，getUnShownContentIdsForSQL 会返回空字符串

迁移步骤

  1. 将原来的 getUnShownContentIdsForSQL 方法中的逻辑迁移到 AITaskDataManager
  2. 在 RecyclerView 中注册数据到 AITaskDataManager
  3. 修改 startAITaskProcedure 方法，使用 AITaskDataManager 获取数据
  4. 在需要的地方直接通过 taskKey 调用 AI 任务

这样就实现了与 RecyclerView 的完全解耦，可以在任意地方通过 taskKey 调用 AI 任务。
