最近，在卷一个我觉得意义不大、没什么用、但又有点技术含量的事情，fp 回溯

为什么要搞呢？

核心原因，并不是大家对于 fp unwind 的普遍认知：速度快。而是，缩减包体积···

是的，没错···缩减···包体积

为什么要缩减包体积呢？

老板的业绩

为什么能缩减包体积呢？

因为 unwind table 比较大，fp unwind 又不依赖于 unwind tables，那么就可以把 unwind table 删掉，即：又快又省空间

同时，既然 fp unwind 不依赖 unwind table，那它总要依赖点什么吧？不然怎么回溯呢？

依赖寄存器~~

所以开启了 fp unwind 后，arm64 下(本文均讨论的 aarch64 架构下的情况)，x29 寄存器会被占用，x30?(一直是lr？没细研究，总之至少占用一个寄存器)

且在开启一定等级的优化参数以后，fp 是默认会被删掉的，需要强制保留 -fno-omit-frame-pointer

fp 回溯的原理

如图：

fp 和 lr 会像是链表一样，链接所有的栈帧

如何连接的呢？

fp 存储的值为 caller fp 的地址，fp 和 lr 存储在连续的16字节空间中，那么我们就可以根据这条链路，不断向上回溯，拿到所有的 fp 和 lr，进而也就是我们的堆栈

这就是核心原理了，就是这么简单

但是，仔细想想，又会发现不对。既然不断向上回溯，那么中止条件是什么呢？

thread stack 边界

中止条件，就是有没有越出 stack 边界，我们可以使用如下方式获取 stack 的边界

pthread_attr_t attr;
pthread_getattr_np(pthread_self(), &attr);
thread_stack_high = (uintptr_t) (attr.stack_base) + attr.stack_size;
thread_stack_low = (uintptr_t) (attr.stack_base);

遍历的时候 check 边界即可，参见 bionic libc 中的 fp unwind 的简单实现：

__attribute__((no_sanitize("address", "hwaddress"))) size_t android_unsafe_frame_pointer_chase(
    uintptr_t* buf, size_t num_entries) {
  // Disable MTE checks for the duration of this function, since we can't be sure that following
  // next_frame pointers won't cause us to read from tagged memory. ASAN/HWASAN are disabled here
  // for the same reason.
  ScopedDisableMTE x;

  struct frame_record {
    uintptr_t next_frame, return_addr;
  };

  auto begin = reinterpret_cast<uintptr_t>(__builtin_frame_address(0));
  auto end = __get_thread_stack_top();

  stack_t ss;
  if (sigaltstack(nullptr, &ss) == 0 && (ss.ss_flags & SS_ONSTACK)) {
    end = reinterpret_cast<uintptr_t>(ss.ss_sp) + ss.ss_size;
  }

  size_t num_frames = 0;
  while (1) {
#if defined(__riscv)
    // Frame addresses seem to have been implemented incorrectly for RISC-V.
    // See https://reviews.llvm.org/D87579. We did at least manage to get this
    // documented in the RISC-V psABI though:
    // https://github.com/riscv-non-isa/riscv-elf-psabi-doc/blob/master/riscv-cc.adoc#frame-pointer-convention
    auto* frame = reinterpret_cast<frame_record*>(begin - 16);
#else
    auto* frame = reinterpret_cast<frame_record*>(begin);
#endif
    if (num_frames < num_entries) {
      uintptr_t addr = __bionic_clear_pac_bits(frame->return_addr);
      if (addr == 0) {
        break;
      }
      buf[num_frames] = addr;
    }
    ++num_frames;
    if (frame->next_frame < begin + sizeof(frame_record) || frame->next_frame >= end ||
        frame->next_frame % sizeof(void*) != 0) {
      break;
    }
    begin = frame->next_frame;
  }

  return num_frames;
}

细节问题

上述的 bionic libc 用于普通的回溯是没有问题的

但是，当我们处于 crash 进行回溯时，就不是这么简单了

首先，我们需要把 signalhandler 返回的 context 中的 pc 值，fp值取到

其次，我们需要获取 signal stack 的范围，然后再切换到 thread stack

最后，我们的 lr 地址要 -4，因为 lr 地址是 bl 指令帮忙填充的，指向了下一条 arm 指令，而如果是 noreturn 函数且就一行的话，就会发生堆栈解析出错的情况

比如 function1 -> function2 -> function3，会变为 function1 -> function3

且上述逻辑只是在解析函数地址，地址拿到了以后还要去解析函数符号，才能是完整的堆栈

目前，我司的 crash sdk 为 xcrash，xcrash 内部又使用了 fork + execl，导致情况更加复杂，需要进行跨进程通信

fp 回溯的实现

static size_t xcc_fp_unwind_custom(uintptr_t *frames, size_t frames_cap, void *context) {
    // 初始化 thread stack 区间
    uintptr_t begin = thread_stack_low;
    uintptr_t end = thread_stack_high;
    uintptr_t cur_fp;
    size_t num_frames = 0;

    if (context == NULL) {
        cur_fp = (uintptr_t) __builtin_frame_address(0);
    } else {
        mcontext_t *mc = &(((ucontext_t *) context)->uc_mcontext);
        cur_fp = mc->regs[29];
        // 先 append pc
        frames[num_frames++] = mc->pc;
    }

    stack_t ss;
    bool unwind_on_signal_stack = false;
    // 初始化 signal stack 区间
    if (sigaltstack(NULL, &ss) == 0 && (ss.ss_flags & SS_ONSTACK)) {
        begin = (uintptr_t) ss.ss_sp;
        end = (uintptr_t) (ss.ss_sp) + ss.ss_size;
        unwind_on_signal_stack = true;
    }

    while (num_frames < frames_cap) {
        frame_record_t *frame = (frame_record_t *) (cur_fp);
        uintptr_t next_fp = frame->next_frame;
//        if (unwind_on_signal_stack &&
//            (frame->return_addr != 0 &&
//            // __kernel_rt_sigreturn in vDSO:
//            // 0xd2801168   mov x8, #0x8b
//            // 0xd4000001   svc #0x0
//             (0xd4000001d2801168ULL == *((uint64_t *) frame->return_addr)))) {
//            frames[num_frames++] = clear_pac_bits(frame->return_addr);
//        } else {
        frames[num_frames++] = clear_pac_bits(
                frame->return_addr > 4 ? frame->return_addr - 4 : 0);
//        }
        // signal stack -> thread stack 边界转换
        if (unwind_on_signal_stack &&
            (frame->next_frame < begin + sizeof(frame_record_t) || frame->next_frame >= end)) {
            unwind_on_signal_stack = false;
            begin = thread_stack_low;
            end = thread_stack_high;
        }
        // 确保边界
        if (frame->next_frame < begin + sizeof(frame_record_t) || frame->next_frame >= end ||
            frame->next_frame % sizeof(void *) != 0) {
            break;
        }
        // 确保顺序
        if (next_fp < cur_fp + sizeof(frame_record_t)) {
            break;
        }
        cur_fp = frame->next_frame;
    }
    return num_frames;
}
