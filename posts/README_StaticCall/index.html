静态调用 getUnShownContentIdsForSQL 方法

概述

原来的 getUnShownContentIdsForSQL 方法需要依赖 VGWidgetScrollDelegate 实例才能调用。现在通过 AITaskDataManager 的静态方法，可以在任意地方直接调用，完全解耦。

解决方案

1. 静态方法调用

// 直接静态调用，不需要任何实例
val sqlResult = AITaskDataManager.getUnShownContentIdsForSQL(data, position)

2. 可用的静态方法

// 获取SQL格式的未显示内容ID列表
AITaskDataManager.getUnShownContentIdsForSQL(data, position)

// 获取原始列表（不包含SQL格式）
AITaskDataManager.getUnShownContentIds(data, position)

// 将内容ID列表转换为SQL IN子句格式
AITaskDataManager.convertToSQLInClause(contentIds)

使用场景

1. 在任意业务逻辑中调用

class SomeBusinessLogic {
    fun processVideoData(data: VideoGoodsViewListData, position: Int) {
        // 直接调用，不需要依赖任何实例
        val contentIds = AITaskDataManager.getUnShownContentIdsForSQL(data, position)
        
        // 使用结果
        val sql = "SELECT * FROM content WHERE id IN ($contentIds)"
        println("构建的SQL: $sql")
    }
}

2. 在AI任务中使用

val featureProvider = object : FeatureProvider<FloatArray> {

    }
}

优势

  1. 完全解耦：不依赖任何实例，可以在任意地方调用
  2. 线程安全：静态方法天然线程安全
  3. 易于测试：不需要创建复杂的实例来测试
  4. 代码简洁：调用方式简单直接
  5. 性能优化：避免了不必要的对象创建

迁移指南

原来的调用方式

// 需要创建实例
val delegate = VGWidgetScrollDelegate(...)
val sqlResult = delegate.getUnShownContentIdsForSQL()

新的调用方式

// 直接静态调用
val sqlResult = AITaskDataManager.getUnShownContentIdsForSQL(data, position)

注意事项

  1. 确保传入的 data 和 position 参数正确
  2. 方法会过滤掉直播内容（mediaType 为 “LIVE” 的内容）
  3. 只返回未显示且 contentId 不为空的内容
  4. 返回的SQL格式为 'id1','id2','id3'

示例输出

// 输入
val data = VideoGoodsViewListData() // 包含多个视频数据
val position = 5

// 输出
val result = AITaskDataManager.getUnShownContentIdsForSQL(data, position)
// 结果: "'content_6','content_7','content_8'"
