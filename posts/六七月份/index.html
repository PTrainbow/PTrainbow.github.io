前言

距离上一次的 Coroutines 的文章已经过去了一个半月了(这个文章还有点小尴尬，其实我也只是写了几个小 demo，验证了一些自己的观点罢了，其实也没有实际的使用经验，然鹅经常被人问，我最近又瞄了一眼文档，似乎 select 也要出来了，这岂不形式上就是 go 的那一套东西了？但是好像比 go 理解起来又复杂了很多很多)

这一个半月我回头仔细想了想干啥了？

工作上似乎都交给适配鸿蒙了，生活上看了很多非计算机类的书，比如 《洞察人性》、《次第花开》 甚至看起了 《周易》，开始背八卦，乾三连，坤六断，震仰盂，艮覆碗~~

都是一些心理学和宗教哲学的书

为什么看这些书呢？因为，实在想不通现在的人为啥会这样？看不惯很多事情，究竟是我的问题还是别人的问题？

大的就不说了，价值观啥的，我一个月都喷不完

就举个在公司下楼梯的小例子：

楼梯道就两个人半人的宽度吧，有人上有人下，先不说靠不靠右行吧，这也无所谓了。大部分时间是没啥问题的，但是每当吃饭的点儿，就很尴尬了，被堵在楼梯道上都堵了好几次了。
帅哥或者美女们，低头玩着手机，或者相互聊着天，两人一排，占满楼梯道。
我跟他们说，“不好意思，借个过”，有时候他们都不鸟我，这真的很夸张是不是，但这就是事实，离谱，真是太难了···
后来我就不说话了，默默的下楼，不让我，我也下，如果是个妹子还能撞一下，我可不吃亏。
但是，好玩的就是，一旦你硬气了，他们就软了，反而会避让你···
很多人说，这就是善良的代价，要学着变坏。其实这啥也不是，只是两个人的品性完全不同罢了，这能称得上是什么人生哲理么？害。

鸿蒙 FA 卡片

有关鸿蒙的适配，算是我最近最有收获的、技术相关的东西了吧！当然，其实也没啥收获，如果以后只能用鸿蒙的话，我算是先行了一步吧。

就我写鸿蒙 FA 卡片的一个月以来，感觉鸿蒙就是一个赶出来的、但是又不得不出来的东西。各种 Android 的 API 换了名字，官网的文档比 Weex 的文档还可怕，堪称文档界的鬼才。

还好一直有个微信群，华为大哥有问必答，点个赞，而且感觉他 Android 方面的基础知识应该是比我扎实的。我工作这么多年，接触的业务场景使用到的 Android 的组件，一个手就数过来了，其他的并不了解。

我感觉鸿蒙最好的改进是布局层级拍扁了，只有一层。但是似乎现在都这么玩了，compose 也是一层，估计都是直接从 RenderNode 搞起？

鸿蒙吐槽

  • 系统本身有很多 Bug
    比如 remoteview 的 一些方法直接失效等类似的方法直接失效的问题(感觉是他们有了自己的一套 UI 组件，映射到 Android 组件？可能一些东西没有适配好？)

  • IDE 布局解析器有很多 Bug
    不支持各种 xml 标签，但是因为底层还是 Android，报错，却能通过编译且正常运行。是的，很尬。

鸿蒙好的地方

  • FA 卡片我觉得还挺好的
    虽然就是 shorcuts + widget，但是最起码是创新啊，Android 是没有这东西的(虽然可以做出来，但是也得 framework 层支持)

  • 跨设备的各种功能(高大上就是 分布式 的各种功能)
    这个似乎是建立在华为自己的一套 IPC 里面的，可以做一些页面流转。最常见的就是，手机上看的东西可以转到 PC 端或者电视端等等。

刷算法题

本来是每天刷2道算法题，结果，由于上周太忙了，上周总共就刷了1题，后面继续努力吧！

对于二叉树相关的，常用 递归 的题目，似乎理解的更深刻

学到了一些 算法，比如 快速幂 和 字典序

快手和看书

和前面说的一样，我看了好多心理学和哲学的书籍，主要就是给自己一个解脱，让我能理解，为什么我在快手就是不太开心，为什么身边有这么多看不惯的东西

不知道到底是我的问题，还是同事的问题，还是快手的问题，还是社会的问题

看了一些书后，我姑且认为是，快手的问题吧。我也更加释然且无所谓了。

就保持一个乐观积极的心态就完全 OK 呀

结语

就是因为太久不写东西了，博客空空的，随便说两句
